# panteparak-skills

A personal Claude Code **plugin marketplace** of self-authored development
skills — packaged as **one plugin per skill** so each repository can enable
exactly the skills it needs, with nothing installed globally.

- **Source of truth:** the `SKILL.md` files under [`plugins/`](plugins/).
- **Manifests** (`marketplace.json`, per-plugin `plugin.json`) are generated by
  [`scripts/generate-manifests.py`](scripts/generate-manifests.py) — never edit them by hand.
- **Inventory** of what was previously installed (incl. external skills/plugins
  and their reinstall sources) lives in [`INVENTORY.md`](INVENTORY.md).

## Per-repo setup (no global install)

Add a `.claude/settings.json` to the repo and enable just the plugins you want.
Opening the repo in a fresh session registers the marketplace and auto-installs them:

```json
{
  "extraKnownMarketplaces": {
    "panteparak-skills": {
      "source": { "source": "github", "repo": "panteparak/skills" }
    }
  },
  "enabledPlugins": {
    "dev-go@panteparak-skills": true,
    "test-unit@panteparak-skills": true,
    "git-pr@panteparak-skills": true
  }
}
```

Ready-made starting points per stack are in [`examples/presets/`](examples/presets/)
(`go`, `springboot`, `nextjs`, `k8s`) — copy one to a repo's `.claude/settings.json`
and trim/extend the `enabledPlugins` list.

## Adding or editing a skill

1. Create/edit `plugins/<skill>/skills/<skill>/SKILL.md` (YAML frontmatter with
   `name` + `description`, then the skill body).
2. Regenerate manifests:
   ```bash
   python3 scripts/generate-manifests.py
   ```
3. Validate and commit:
   ```bash
   claude plugin validate .
   git add -A && git commit -m "Add/update <skill>"
   ```

## Use globally (optional)

These are designed for per-repo use, but you can add the marketplace for
personal/global use in a session:

```bash
/plugin marketplace add panteparak/skills
/plugin install dev-go@panteparak-skills
```

## Reinstalling previously-used external skills/plugins

The marketplace plugins and third-party standalone skills that used to be
installed globally are catalogued with their source repos in
[`INVENTORY.md`](INVENTORY.md). Reinstall selectively from there.
